Data and Ontology Analysis

Dataset: Air Quality in Madrid (2024)

This document shows the analysis of the chosen dataset. It includes the data structure investigation, licensing evaluation, and the naming strategy that will be applied to the ontology and RDF data. The dataset's from the Madrid City Council Open Data Portal and contains daily gathered data of air quality indicators such as NO₂, PM₁₀, O₃, SO₂, and CO collected across different monitoring stations in Madrid during 2024.

1. Dataset Structure Analysis

The dataset is provided in CSV format and contains the following key columns:

Column Description Example Value Data Type Notes
ESTACION Code identifying the air quality monitoring station. 4 Integer Links to real-world stations; can be connected to Wikidata or OSM.
PUNTO_MUESTREO Station code including the district and instrument details. 28079004 String Acts as a unique identifier for a specific measurement point.
MAGNITUD Pollutant identifier (e.g., 8 = NO₂, 9 = O₃). 8 Integer Defines the type of pollutant measured.
FECHA Date of the measurement. 2024-05-03 Date Used to track measurements over time.
VALOR Average daily value of the pollutant in µg/m³. 45 Float Primary measurement value.
UNIDAD Unit of measurement (typically µg/m³). µg/m³ String Could be linked to an ontology of measurement units (e.g., OM).

The dataset is clean and mostly structured, with very few missing values.

2. Licensing Analysis

The dataset is published under the Madrid City Council Open Data License, which allows reuse for both commercial and non-commercial purposes as long as you provide attribution. It's compatible with the Creative Commons Attribution 4.0 (CC BY 4.0) license.

  • Rightsholder: Ayuntamiento de Madrid (Madrid City Council)
  • License name: Licencia de uso de los conjuntos de datos del Ayuntamiento de Madrid
  • License URL: https://datos.madrid.es/egob/catalogo/aviso-legal
  • Authorized uses: Copying, distribution, modification, adaptation, and combination of the data.
  • Obligations: Must cite the source as “Origen de los datos: Ayuntamiento de Madrid”; must not distort or misrepresent the meaning of the data; and may not imply that the City Council supports the reuse.
  • Exclusion of responsibility: The City Council is not liable for damages or losses resulting from data reuse.

The generated Linked Data will include the required metadata, maintaining license compatibility with other open datasets.

3. Resource Naming Strategy

The URIs for the ontology and dataset will follow the recommendations provided in the Methodological Guidelines for the Generation of Linked Data. The naming strategy uses both hash and slash URIs depending on the resource type. The domain used below (http://example.org/group07/) is an example placeholder.

  • Base domain: https://carloscumbrado.github.io/HandsOn/group07/
  • Ontology path: ontology/airquality# (for classes and properties)
  • Resources path: resource/ (for individuals)
  • Example ontology term: https://carloscumbrado.github.io/HandsOn/group07/ontology/airquality#AirQualityObservation
  • Example individual: https://carloscumbrado.github.io/HandsOn/group07/resource/observation/2024-05-03-04

4. Conclusions

The Air Quality in Madrid (2024) dataset meets all requirements for Linked Data generation. It will be transformed into RDF using the defined ontology, modeling air quality observations, pollutants, and stations. The resource naming strategy guarantees URI persistence with other smart city datasets.
