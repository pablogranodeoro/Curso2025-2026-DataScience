2. Análisis de licencia

El origen del dataset es OpenAQ, que recopila datos de la European Environment Agency (EEA). La EEA publica sus datos generalmente bajo licencia Creative Commons Attribution 4.0 (CC BY 4.0). Por tanto, se propone aplicar la misma licencia a los datos RDF generados, citando siempre a la EEA y a OpenAQ como fuentes originales.

3. Estrategia de nombrado de recursos (URIs)

Dado que el dataset es pequeño, se opta por usar hash URIs.

  • Dominio base: http://linkeddata.grup05.es/ — bajo control del grupo.
  • Subcarpeta del proyecto: CalidadAireMadrid para indicar el contexto de los datos.

3.1 Ontología

Las clases y propiedades de la ontología tendrán la forma:

  • Clases: http://linkeddata.grup05.es/CalidadAireMadrid/ontology/<ontologyName>#<className>
  • Propiedades: http://linkeddata.grup05.es/CalidadAireMadrid/ontology/<ontologyName>#<propertyName>

Ejemplos:

  • Clase de medición: http://linkeddata.grup05.es/CalidadAireMadrid/ontology/AirQualityOntology#Medicion
  • Propiedad “valor”: http://linkeddata.grup05.es/CalidadAireMadrid/ontology/AirQualityOntology#valor

3.2 Instancias

Las instancias de los datos seguirán el patrón:

  • http://linkeddata.grup05.es/CalidadAireMadrid/resource/<className>/<identifier>

Ejemplo: una medición de NO₂ tomada el 2025-10-27T01:00:00Z en la estación 4320 podría representarse como:

http://linkeddata.grup05.es/CalidadAireMadrid/resource/Medicion/4320_2025-10-27T01:00:00Z_no2

El análisis y la estrategia de nombrado propuestos siguen las buenas prácticas de Linked Data. Se define una ontología ligera y un esquema de URIs consistente con el ejemplo BECA, asegurando claridad, reutilización y atribución correcta según la licencia CC BY 4.0.
