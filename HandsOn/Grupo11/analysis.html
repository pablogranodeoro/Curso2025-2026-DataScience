ANALYSIS OF THE “MADRID COMMUTER RAIL” DATASET

1. Dataset Analysis

The selected dataset contains information about Madrid's commuter rail stations, published by Renfe-Operadora on the open data portal datos.gob.es.

The data is available in CSV and XLSX formats, with a total of 94 records (one row per station).

The fields it contains are as follows:

  • CODE (integer): Unique identifier of each station.
  • DESCRIPTION (string): Official name of the station.
  • LATITUDE (decimal): Geographic coordinate (latitude WGS84).
  • LONGITUDE (decimal): Geographic coordinate (longitude WGS84).
  • ADDRESS (string): Station postal address.
  • ZIP CODE (string): Station postal code.
  • POPULATION (string): Location where the station is located.
  • PROVINCE (string): Corresponding province.
  • PDF SHEETS (anyURI): Link to the station's information PDF sheet.
  • ELEVATED TUNNELS (anyURI): Link to the PDF sheet if it corresponds to an elevated tunnel (only one case).

The information sheet contains the following details:

  • Geographic coordinates in UTM and WGS84 format.
  • Municipality and business line.
  • Operating, contact, and service information.

3. Resource Naming Strategy

The following URI strategy is defined:

  • Base domain: http://group11.linkeddata.es/
  • Ontology: http://group11.linkeddata.es/ontology/station#
  • Stations: http://group11.linkeddata.es/resource/station/{code}
  • Official PDF: http://group11.linkeddata.es/resource/station/{code}/officialPDF
  • Technical facilities: http://group11.linkeddata.es/resource/facility/{code}
